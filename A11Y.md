# Accessibility

## Keyboard Navigation

- Left/Right arrow keys navigate between slides in both edit and
  preview modes
- All interactive elements (buttons, inputs) are accessible via Tab key
- Modal dialogs can be closed with Escape key (built into shadcn Dialog)
- Enter key submits the login form

## Semantic HTML

- Headings (h1, h2) used for page and presentation titles
- Native button elements used for all actions, not divs
- Images include alt text descriptions provided by users
- Form inputs have associated Label components

## Visual Indicators

- Disabled arrow buttons have reduced opacity (30%) to indicate
  non-interactive state
- Selected elements have a visible blue border
- Current slide is highlighted with blue ring in slide list
- Interactive elements have hover states for discoverability
- Active background type button uses filled style vs outline

## Color and Contrast

- High-contrast text (black on white) throughout the UI
- Error messages use red destructive variant for clear visibility
- Slide numbers visible in bottom-left corner of each slide
- Modal overlays use semi-transparent black background (40-50%)
